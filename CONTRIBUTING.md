# Contributing to Balder Brain V5

Thank you for your interest in contributing to Balder Brain V5! We welcome contributions from the community to help improve the performance and reliability of small LLMs.

## How to Contribute

1.  **Report Bugs**: If you find a bug, please open an issue describing the problem and how to reproduce it.
2.  **Suggest Enhancements**: Have an idea for a new feature? Open an issue to discuss it.
3.  **Pull Requests**: 
    - Fork the repository.
    - Create a new branch for your feature or bug fix.
    - Submit a pull request with a clear description of your changes.

## Areas for Contribution

- **Comparison Reasoning**: As noted in our research, this is a current weak point.
- **Latency Optimization**: Making the framework even faster on edge devices.
- **New Benchmarks**: Adding more diverse test cases to our test suite.

## Code Style

Please follow standard Python (PEP 8) coding conventions and ensure your code is well-documented with docstrings.
