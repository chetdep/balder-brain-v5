## 2. Decoupled Responsibility: "Auto-Serializer" (2-Step Process)

To optimize performance on ultra-small language models (such as `gemma4:e4b` operating under 4GB RAM), the system adopts a **2-Step Strategy**:

1. **Step 1 (LLM Reasoning): Model Liberation**
   - Completely removes the requirement for the LLM to output complex JSON blocks (`TurnRoutePlan`).
   - The LLM focuses solely on logical thinking (`Thought`) and specifying the execution tool (`Action`). Token generation speed is maximized, and the risk of JSON syntax errors is reduced to zero.

2. **Step 2 (Code Serializer): Python-Based JSON Packaging**
   - The `ContextEnricher` (Layer 1 Classifier) pre-analyzes the intent (`IntentType`) using Heuristic/Regex algorithms from raw text.
   - When the LLM returns an `Action`, the Python system (Agent Core) automatically synthesizes the Intent and Action to **Auto-Generate** the JSON routing plan into Telemetry. The system achieves absolute structural precision.
